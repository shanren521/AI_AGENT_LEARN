# 1. 索引失效的情况

+ 对索引列使用函数 select * from t where date(create_time) = '2026-06-16'; -- 失效
+ 索引字段参与四则运算 select * from t where age + 10 = 30; -- 失效
+ 字段类型和查询值类型不一致，MySQL 自动转类型导致索引失效。
  + select * from t where phone = 13800138000; -- 失效 idx_phone(phone varchar(11))
  + select * from t where id = '123'; -- 不会失效，数字转字符串不影响 idx_id(id int)
+ 模糊查询 % 前缀通配 
  + select * from t where name like '%张三'; -- 失效，全表扫描 
  + select * from t where name like '张三%'; -- 生效，前缀匹配可用B+树 
  + select * from t where name like '%张三%'; -- 失效
+ or 左右条件有非索引列
  + select * from t where a=1 or b=2; -- 索引失效，全表扫描 idx_a(a), b无索引
  + 解决方案 - union 拆分两条索引查询: select * from t where a=1 union select * from t where b=2;
+ 联合索引违反最左前缀原则：联合索引 idx_ab(a,b,c)，查询条件必须匹配最左前列，否则无法走索引。
  + where b=1; -- 缺a，失效 
  + where c=2; -- 缺a、b，失效 
  + where b=1 and c=2; -- 缺a，失效
+ not != <> !is null 否定查询：不等于、非空判断大多放弃索引走全表
  + where status != 1; -- 失效 
  + where age <> 20; -- 失效 
  + where name not like '李%'; -- 失效 
  + where id not in (1,2,3); -- 失效 
  + where create_time not null; -- 失效
+ in /exists 数据量过大 
  + in 集合数量上千，MySQL 优化器认为扫描索引不如全表，放弃索引
  + 子查询返回超大结果集，exists 无法利用外层索引。
+ order by 索引排序不匹配、filesort -- 联合索引 idx_ab(a,b) (规则：where 过滤字段 + order by 字段必须连续匹配最左前缀，且升降序统一。)
  + where a=1 order by c; -- 无索引，文件排序
  + where a=1 order by b desc,c asc; -- 排序方向不一致也会失效索引排序
+ 关联查询条件类型不匹配（join 失效） 
  + 两表 join 关联字段类型不一致，触发隐式转换，关联索引失效：
+ force index 以外强制忽略索引操作
  + ignore index(idx) 显式禁用索引；
  + 使用 select * 大量回表，优化器放弃二级索引直接全表；
  + 分区表未指定分区，跨分区查询无法走局部索引。

# 2. Redis 雪崩、击穿、穿透

+ 缓存雪崩
  + 原因：大量缓存 key 同一时间集体过期失效，或 Redis 集群整体宕机，大量请求瞬间全部打到数据库，数据库 CPU / 连接池打满、卡死、宕机，引发连锁崩溃。
  + 解决方案：
    + 过期时间随机打散（解决批量过期）
    + Redis 高可用架构（解决集群宕机）主从 + 哨兵 / Redis Cluster 集群，故障自动切换；
    + 多级缓存：本地 JVM 缓存（Caffeine/Guava）+ Redis 二级缓存，Redis 挂了本地兜底；
    + 限流 & 熔断降级：网关 / 服务层限流（Guava RateLimiter、Sentinel），DB 请求过载直接返回兜底数据；
    + 集群扩容 分摊请求压力。
+ 缓存击穿
  + 原因：某个热点高频访问的 key 过期瞬间，大量并发请求同时失去缓存，并发穿透到数据库查询同一条数据。
  + 解决方案：
    + 互斥锁（分布式锁）：缓存失效时，只放行一个线程去查 DB、回填缓存，其他线程阻塞等待缓存重建；
    + 逻辑永不过期（热点 key 专用，推荐）
    + 缓存预热 + 提前续期
    + 本地缓存兜底：JVM 缓存存一份副本，Redis 过期瞬间本地临时承接流量。
+ 缓存穿透
  + 原理：查询数据库根本不存在的数据，缓存中永远不会有这条 key，请求每次都会绕过缓存直达数据库。
  + 解决方案：
    + 缓存空值（最简单通用）：DB 查询为空时，向 Redis 存入空标记，设置短过期（5 分钟）
    + 布隆过滤器 Bloom Filter（防攻击最优）：提前把所有合法有效 ID 存入布隆过滤器，查询前先过过滤器。
    + 参数校验拦截：接口层提前校验 ID 范围、格式，非法参数直接拦截，不进缓存逻辑；
    + 限流防护：单 IP 短时间大量未命中请求直接限流封禁，抵御爬虫 / 攻击。
